

在为程序分配内存后，OS会对内存进行划分，划分的大致过程：
```
1.首先内存被大致划分为数据区和代码区，其中代码区又称为[代码段]或者文本段
2.数据区进一步划分为动态数据区和静态数据区
3.静态数据区又称为[数据段]，进一步划分为三个分段：.bss段,.data段,.rodata段
4.动态数据区又被划分为两个区域：[堆区]和[栈区]
```

传统的操作系统设计和开发中通常归为四个区域：代码段code segment、数据段data segment、堆区heap、栈区stack
```
内存进一步的详细布局：虚拟内存空间中从低到高依次为.text、.rodata、.data、.bss，紧接着是heap，然后stack位于栈顶

代码段和数据段在编译链接期间就确定大小，程序加载时分配空间，运行时固定不变，由编译器自动管理，开发者无需也无法干预
堆区和栈区只在程序运行的期间再进行内存的分配和管理，栈区/堆区分别自上而下/自下而上使用内存，二者动态共享中间的内存


所以程序内存布局 = 静态（代码段+数据段） + 动态（堆+栈）
- 静态：运行前分配，结束后释放
- 动态：运行时分配，运行时释放
注意这里的动态和静态指的是内存分配的时机，一个是运行前一次性分配运行结束后统一释放，一个是运行期间再进行分配和释放
```

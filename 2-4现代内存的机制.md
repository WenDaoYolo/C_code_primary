
GNU
```
linux只是一个开源的系统内核，负责资源分配和调度等核心操作，但其本身并不能构成完整的操作系统
而早期Unix操作系统是闭源的，用户不能真正彻底的掌握计算机，于是就出现了一个名为GNU的自由项目

GNU's Not Unix，意为GNU并非Unix，它最初理念就在于自由和开源，GNU系统所有的工具环境都已开
发完成，仅缺一个内核，之后Linux内核作为自由软件发布，其他人则将GNU系统和Linux相结合创造了
首个完整的开源操作系统GNU/Linux，因为开源所以全球人都可以共同参与系统的维护和升级，并且由此
衍生出了许多其他版本，例如ubuntu，centos，kail-linux等，可在发展过程中GNU/linux系统逐渐
演变为linux操作系统，但我们应该严谨和尊重，他是GNU和Linux共同组成的开源操作系统GNU/linux
```

操作系统生态与实验环境
```
GNU/linux是开源的操作系统，其内部的细节以及源码被剖析的很清楚，而其他操作系统都采用不同的内核和架构
例如windows，macos，这一类操作系统都是商业化产品并没有过多对外开源或公开细节，虽然大体框架和机制都
可能相似，但其内部细节和机制我们无法也无需研究，所以学术界计算机逻辑系统的研究大多以GNU/linux为标准
```

因安全性和实用性的提升，现代操作系统在传统内存模型上增加了许多的新机制


内存映射段(Memory Mapping Segment)
```
是栈区和堆区之间的一个独立区域，运用虚拟内存技术，主要用于进程之间内存的映射与共享，采用按需分页、惰性加
载的方式，其内存大小、分配以及相关内核操作均在程序运行期间由OS完成(mmp/munmp系统)，但可手动提前释放内存
```

工作原理与特性
```
例如动态库的映射与共享(详见编译链接篇)
机器码不会写入可执行文件中，只记录相关元信息，在运行期间首次只会预先分配逻辑页，调用所需的动态
库函数时，会访问虚拟地址并触发缺页中断，此时才实际分配物理页并建立映射关系，然后操作系统更新页
表MMU完成地址转换，因为不写入所以也依赖于这个库文件，运行期间需要不断查询，多个进程可以映射同
一个动态库机器码(即动态库文件共享)，其中内存的共享机制以及惰性加载的方式都大大节省了内存空间

例如进程之间的通信，通过映射共享同一内存区域，让多个进程之间可以相互通信并产生联系
```

现代操作系统的安全机制：ASLR(Address Space Layout Randomization)，PIE(Position Independent Executable)
```
早期设计中因内存有限，为节省内存将程序模型设计的更紧凑，导致程序各段地址都是固定的
首先在安全上就造成了很大的隐患，非法入侵者可以依靠固定的地址从而完成系统程序的入侵
其次只能运行比内存小的程序，否则内存不够用，堆区和栈区相向生长，碰撞后则会程序崩溃

现代操作系统中因为内存非常庞大，各段之间有足够大的隔离带，所以几乎不可能会发生碰撞
于是出现了ASLR和PIE机制，总体上各段的排列保持不变，但每个段在划定的局部范围内其地
址空间随机化，每次运行程序时都会动态随机分配各段地址，从而让非法入侵者很难找到地址
```

安全机制的依赖关系
```
ALSR=0级             ->   关闭随机化
ASLR=1/2级           ->   开启stack,heap的随机化
ASLR=2级 + 启用PIE    ->   开启stack,heap,data_segment,code_segment的随机化
```

ASLR的修改：配置内核参数 
```
内核配置文件的路径：root/etc/sysctl.conf
首次打开系统时具有默认行为：默认开启等级2，添加或修改配置后不再具有默认行为

kernel.randomize_va_space=0/1/2
```

PIE开启与关闭：配置编译命令
```
编译时默认开启PIE

g++/gcc [-no-pie/pie] YourProgram...   
```

代码验证(GNU/Linux环境)
```
#include<iostream>
#include<iomanip>
using namespace std;

int b5;               //.bss
int b3=10;            //.data
const int b4=1;       //.rodata 

void stack_fun()
{
    int b2;           //stack
    int* b1=new int;  //heap

    cout<<left<<setw(20)<<"stack address:"<<hex<<&b2<<endl; 
    cout<<left<<setw(20)<<"heap address:"<<hex<<b1<<endl;
    
    delete b1;
    b1=NULL;
}

int main(){
    stack_fun();
    cout<<left<<setw(20)<<"bss address:"<<hex<<&b5<<endl;
    cout<<left<<setw(20)<<"data address:"<<hex<<&b3<<endl;
    cout<<left<<setw(20)<<"rodata address:"<<hex<<&b4<<endl;
    
    return 0;
}
```
